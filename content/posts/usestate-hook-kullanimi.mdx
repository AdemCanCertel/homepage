Sayı durumuna sahip bileşeniniz olduğunu ve aynı durumu kullanmanız için başka bir bileşene aktardığınızı hayal edin: 

```javascript
import { useState } from 'react'
import { Snippets } from './Card/Snippets'

export const Sample = () => {
    cosnt [ number, setNumber ] = useState<number>(0)

    return (
        <Snippets number={number} setNumber={number} />
    )
}
```

SnippetsProps arayüzünü nasıl görünmelidir?
Çoğu insan şöyle yazıcaktır:

```javascript
import { Dispatch, SetStateAction } from 'react'

interface SnippetsProps {
    number: number
    setNumber: Dispatch<SetStateAction<number>>
}

export const Snippets = ({ number, setNumber }: SnippetsProps) => {
    return(
        <div>
            <span>
                Clicked { number } times
            </span>
            <button
                onClick={() => setNumber(number + 1)}> Click me
            </button>
        </div>
    )
}
```

Ancak, sadece şunu yapabilmek için "Dispath" ve "SetStateAction" React'ten almam gerekiyor.

Gönderim "Dispatch<SetStateAction<T>>" türü temel oalrak her türü kabul eden bir işlevdir.

Bu nedenle, burada da yapabilirsiniz: 

```javascript
interface SnippetsProps {
    number: number
    setNumber: (v: number) => void
}

export const Snippets = ({ number, setNumber }: SnippetsProps) => {
    return(
        <div>
            <span>
                Clicked {number} times
            </span>
            <button
                onClick={() => setNumber(number + 1)}>Click
            </button>
        </div>
    )
}
```

Sadece bir nuamrayı kabul eden bir işlev olarak beyan ederek iki tasarruf yaptık.

Bunun sadece küçük bir ayrıntısı olduğunu biliyoruz, ama gerçekten bu kodun daha temiz görünmesini sağlıyor.
